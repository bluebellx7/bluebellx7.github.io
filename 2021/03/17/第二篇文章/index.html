Ø OpenSSL SSL_read: Connection was reset, errno 10054

git config –global http.sslVerify “false”

Ø Git基本配置

git config –list # 配置列表
git config –global user.name “user” # 设置用户名，git log中可以看到是谁提交的
git config –global user.email “user@gmail.com“ # 设置邮箱，可用于发生邮件

Ø 项目Git初始化

🎈 如果已经存在一个demo1 项目
cd demo1 # 切换到demo1项目目录
git init # 把项目初始化为仓库，会在项目目录下新建一个.git目录
🎈 如果不存在项目则
git init demo1 # 会直接创建一个仓库

Ø Git状态

git status # 查看文件状态,红色的为还未添加到暂存区的，绿色的为添加到暂存区的
git add index.html images/ # 把index.html和images文件夹 添加到暂存区
git add -u # 把工作区修改过的所有文件添加到暂存区

git commit -m “提交的备注” # 提交暂存区中所有的更改到仓库
git commit -am “该命令为工作区的内容直接加入暂存区并提交到版本库”

git reset –hard # 清除工作区和暂存区所有内容
git reset –hard HEAD^ # 还原到上个版本；
git reset –hard HEAD^^ # 还原到上上个版本；
git reset –hard HEAD~100 # 还原到上100个版本；
git reset –hard 48d9 # 返回版本到48d9，48d9是commit id(提交commit 时候的id)

git mv readme readme.md # 直接通过git修改文件名，修改后的文件会自动添加到暂存区
git log –all –graph –oneline -n10 # 查看历史，–all 查看所有分支 –graph 比较美观的界面 –oneline 简介一个记录一行 -n10 查看最近10条
git reflog 查看历史提交命令；
git checkout -b temp 9182ar32t2 # 根据log中9182ar32t2 创建temp分支 并切换到temp分支

git checkout master # 切换到master分支
git merge temp #合并temp分支

git config –local user.name “bluebellx7”
git config –local user.email “745423178@qq.com“

git diff 查看修改内容

不需要输入用户名密码

🌹 git config –global credential.helper store

拉取远程分支
git fetch
git branch
git checkout [分支名称]
